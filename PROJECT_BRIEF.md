# Проект: Dragline — каталог б/у спецтехники из Кореи

## Контекст и бизнес-модель

Я строю сайт-каталог б/у экскаваторов, запчастей и другой спецтехники,
расположенной в Южной Корее. Целевая аудитория покупателей — Узбекистан,
Кыргызстан, страны СНГ, дополнительно English-аудитория для расширения охвата.

Бизнес-модель: корейские продавцы (дилеры экскаваторов) выставляют технику
через меня бесплатно. Я (через зарегистрированное в Корее ИП/법인) выкупаю
технику у продавца и организую отправку покупателю. Сайт — это витрина/каталог,
НЕ маркетплейс с онлайн-оплатой. Связь с покупателем — через соцсети
(Telegram/WhatsApp/Instagram), там же консультируем по доставке и растаможке.

## КРИТИЧЕСКИ ВАЖНО — модель данных

На старте НЕТ формы регистрации продавцов и НЕТ формы добавления объявлений
на сайте. Все объявления добавляю Я ВРУЧНУЮ через Google Sheets, который
служит базой данных. Сайт — read-only каталог, который читает данные из
Google Sheets через SheetDB API.

НЕ создавай:
- Систему регистрации/авторизации пользователей
- Backend для записи данных
- Личные кабинеты продавцов
- Корзину, оплату, чекаут

## Технический стек (фиксированный, не менять без согласования)

- Frontend: чистый HTML + CSS + Vanilla JS (НЕ React, НЕ Vue, НЕ фреймворки —
  нужна максимальная простота для деплоя на GitHub Pages и лёгкого редактирования)
- Хостинг: GitHub Pages
- База данных: Google Sheets (я предоставлю Sheet ID после создания)
- API для чтения данных: SheetDB (https://sheetdb.io) — REST API, отдаёт JSON
  из Google Sheets
- Хостинг изображений: Cloudinary (я загружаю фото вручную, в таблицу пишу URL)
- Конвертация валют: бесплатный API типа exchangerate-api.com или
  open.er-api.com (KRW → USD, обновление курса при загрузке страницы)

## Языки сайта (4 шт.)

- 한국어 (ko) — для корейских продавцов, которые могут заходить посмотреть
  как выглядит их объявление
- O'zbekcha (uz) — основная аудитория покупателей
- Русский (ru) — основная аудитория покупателей
- English (en) — для расширения охвата (Африка, Ближний Восток, ЮВА)

Реализация: i18n через отдельные JSON-файлы (locales/ko.json, uz.json,
ru.json, en.json) + переключатель языка в шапке сайта (флаги или коды стран).
Выбранный язык сохраняется в localStorage. Дефолтный язык — определяется по
браузеру пользователя, fallback на English.

## Структура данных (Google Sheets)

Один лист "listings" со следующими колонками:

| Колонка | Тип | Описание |
|---|---|---|
| id | string | Уникальный ID лота (например EXC-001) |
| category | string | excavator / parts / other_machinery |
| brand | string | Марка (Hyundai, Doosan, Komatsu и т.д.) |
| model | string | Модель |
| year | number | Год выпуска (пусто для parts) |
| hours | number | Наработка в часах (пусто для parts) |
| condition | string | Состояние (например "양호", "수리 필요") |
| compatible_models | string | Для category=parts — совместимые модели |
| price_krw | number | Цена в вонах |
| location_kr | string | Локация в Корее (например "경남 창원") |
| photos | string | Список URL фото через запятую (из Cloudinary) |
| status | string | available / sold / reserved |
| name_ko, name_uz, name_ru, name_en | string | Название лота на каждом языке |
| desc_ko, desc_uz, desc_ru, desc_en | string | Описание на каждом языке |

## Структура сайта

### Главная страница (index.html)
- Шапка: логотип/название "Dragline", переключатель языков (4 флага/кода)
- Фильтры: по категории (Экскаваторы / Запчасти / Другая техника),
  по марке, по диапазону цены
- Сетка карточек лотов (грид, responsive — 1 колонка на мобиле,
  3-4 на десктопе)
- Карточка лота: главное фото, название (на текущем языке), марка+модель,
  год+наработка (если применимо), цена в KRW + автоконвертация в USD,
  кнопка "Подробнее"

### Страница лота (listing.html?id=EXC-001)
- Галерея фото (с возможностью переключения/lightbox)
- Полная информация: марка, модель, год, наработка, состояние, локация
- Цена: KRW крупно + USD рядом
- Описание (на текущем языке)
- Блок "Доставка и оформление": текст вида "Стоимость доставки и
  таможенного оформления рассчитывается индивидуально под вашу страну —
  свяжитесь с нами" + кнопки/иконки Telegram, WhatsApp, Instagram со
  ссылками (заглушки на старте, я подставлю реальные позже)
- Кнопка "Назад в каталог"

### Страница "О нас" / "Как это работает" (about.html)
- Краткое объяснение модели: "Мы напрямую сотрудничаем с проверенными
  корейскими продавцами, выкупаем технику и организуем доставку"
- Контакты

## Дизайн

- Чистый, современный, минималистичный стиль — ориентир на промышленную/
  техническую тематику (можно использовать тёмно-синий/жёлтый/оранжевый
  как акцентные цвета — ассоциация со спецтехникой)
- Полностью responsive (мобильная аудитория важна — многие из СНГ
  заходят с телефонов)
- Шрифт с поддержкой кириллицы и латиницы (для узбекского на латинице
  и русского) — например Inter, Roboto, или Noto Sans
- Для корейского текста — убедиться что шрифт поддерживает хангыль
  (Noto Sans KR как fallback)

## Конвертация валют

- При загрузке страницы делать один запрос к exchangerate API за курс
  KRW→USD, кэшировать в localStorage на 1 час (чтобы не делать запрос
  на каждую карточку)
- Если API недоступен — fallback на захардкоженный примерный курс
  (например 1 USD = 1350 KRW, обновляемый вручную в коде как константа)
